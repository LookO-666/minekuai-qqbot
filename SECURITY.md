# 安全策略

## 支持范围

安全修复只针对 `main` 分支的最新版本。

## 报告漏洞

请通过 GitHub 仓库的 **Security → Report a vulnerability** 私下报告。不要使用公开 Issue，也不要提交真实凭据、数据库或登录态文件。

报告中请尽量包含：

- 受影响版本或提交；
- 可复现的最小步骤；
- 潜在影响；
- 已脱敏的日志或示例。

## 凭据泄露

如果凭据已经出现在公开位置，请先在对应平台撤销或轮换，再清理 Git 历史。仅删除最新提交中的文件，不能消除历史泄露。

本项目中尤其需要保护：

- `bot/.env` 和 `CREDENTIAL_ENCRYPTION_KEY`；
- `bot-data/` 下的 SQLite 数据库及备份；
